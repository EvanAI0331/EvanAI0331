# Evan AI

High school student building practical AI developer tools.

I work on Codex workflow frameworks, agent governance, and spec-driven execution systems that make AI agents easier to control, verify, and ship.

## Focus

- Agent workflow frameworks for Codex
- Spec-driven governance and fail-closed verification
- CLI and MCP tools for developer automation
- Research prototypes that turn messy ideas into runnable systems

## Current Projects

- [SpecX](https://github.com/EvanAI0331/specx-codex-plugin): governance layer for Codex agents with contracts, gates, evidence, execution results, and failure semantics.
- [Harness](https://github.com/EvanAI0331/harness-codex-plugin): workflow framework runtime for custom Codex agent systems with specs, artifacts, traces, and verification.

## Principles

- No fake success.
- No silent fallback.
- Every serious agent workflow should be contract-bound, evidence-bound, and failure-explicit.

## Building Toward

I want AI developer tools to feel less like prompts and more like reliable engineering systems: versioned, testable, inspectable, and safe to extend.
